# MangaReaderTranslate

*[Read this in English](README.md)*

Бесплатный, полностью локальный/оффлайн переводчик манги для Windows.
Указываешь страницу манги (или целую папку, или `.cbz`), выбираешь исходный
и целевой язык — и приложение находит текст в каждом облаке диалога,
переводит его, стирает оригинальный текст и вписывает перевод обратно —
всё на твоей собственной машине, без облачных API, без аккаунтов, без платы
за страницу.

![Скриншот](docs/screenshot.png)

## Что оно на самом деле делает

1. **Находит текст** — детектор сканирует страницу и находит каждое облако
   диалога, текстовую вставку и звуковой эффект.
2. **Читает его** — OCR превращает каждый найденный бокс в текст, используя
   специализированный на манге движок для японского и универсальный — для
   остальных ~39 исходных языков.
3. **Переводит** — локальная модель перевода переводит текст на выбранный
   целевой язык (~40 языков, любая пара).
4. **Стирает оригинал** — модель инпейнтинга убирает оригинальный текст с
   картинки, сохраняя линии и растровый тон под ним, так что не остаётся
   размазанного пятна на месте старого текста.
5. **Вписывает перевод обратно** — с переносом строк, автоподбором размера
   под бокс и поворотом под наклон оригинального текста.

Всё работает локально через `onnxruntime`/`ctranslate2`. Приложение само
задействует GPU, если у тебя совместимая видеокарта NVIDIA, и без всякой
дополнительной настройки откатывается на CPU, если нет.

## Возможности

- **Открывай что угодно**: одну картинку, целую папку или архив
  `.cbz`/`.zip`. Перетаскивание файла тоже работает.
- **Добавляй страницы по ходу дела**: уже загрузил страницы и хочешь
  добавить ещё — один файл или целую пачку — без того, чтобы начинать
  заново? `Add Pages` (или просто перетащи ещё файл/папку) добавляет, а не
  заменяет.
- **Авто-разбиение "длинных лент"**: некоторые сайты с сырыми сканами
  упаковывают целую главу в одну огромную склеенную по вертикали картинку
  вместо отдельного файла на страницу. Приложение распознаёт такую форму и
  автоматически разбивает её обратно на отдельные страницы при импорте.
- **Меняй порядок и удаляй страницы**: перетаскивай страницы в списке, чтобы
  поменять порядок; выделяй несколько (Ctrl/Shift-клик) и удаляй выбранные
  или очищай всё сразу прямо из боковой панели.
- **Пакетный или одиночный перевод**: переводи весь загруженный набор в
  фоне с прогресс-баром, или только ту страницу, которую сейчас смотришь.
- **Ручная правка**: кликни по любому найденному боксу на странице (или
  выбери его из списка) — увидишь его исходный текст (OCR) и перевод.
  Отредактируй перевод — и он перерисуется автоматически через мгновение
  после того, как перестанешь печатать. Не нужно заново гонять OCR/перевод/
  инпейнтинг ради одной строки.
- **Свой шрифт на уровне бокса, страницы или всего приложения**: не
  нравится автоматически подобранный шрифт для одного облака? Задай шрифт
  именно для этого бокса. Нужен конкретный шрифт для целой страницы или
  для всех страниц по умолчанию? Settings → Custom Font (на всё
  приложение), или правый клик по странице в списке → Set Font for This
  Page. Более точная настройка всегда побеждает.
- **Zoom, панорамирование и шторка сравнения до/после**: колесо мыши —
  зум (по центру курсора), перетаскивание — панорама, и перетаскиваемая
  шторка, чтобы сравнить оригинал и перевод side-by-side.
- **Листание страниц под себя**: наведи курсор на страницу — появятся
  полупрозрачные кнопки prev/next (расположение зависит от режима чтения,
  см. ниже), либо просто пользуйся стрелками на клавиатуре.
- **Автосохранение сессии**: закрыл приложение (даже случайно) не
  экспортировав результат — при следующем запуске оно предложит
  восстановить переведённые страницы и все ручные правки.
- **Экспорт** готовых страниц как обычных картинок или сразу в новый
  `.cbz`.

## Системные требования

Это не лёгкое приложение — заложи место на диске под ML-модели плюс venv
или собранный exe.

|  | Минимум | Рекомендовано |
|---|---|---|
| ОС | Windows 10/11 (64-bit) | Windows 10/11 (64-bit) |
| Python (из исходников) | 3.11 | 3.11 |
| CPU | Любой 64-bit, 4 ядра | 4+ ядра, не старше ~5-6 лет |
| RAM | 8 ГБ | 16 ГБ |
| GPU | Не нужен — работает на CPU | NVIDIA, 4+ ГБ VRAM |
| Свободное место на диске | ~9-10 ГБ | ~15 ГБ (SSD) |

Примечания:

- **Разбивка по диску**: сами модели (детектор + инпейнтинг + NLLB +
  manga-ocr + RapidOCR) занимают примерно **3.5-4 ГБ** суммарно. Сверх
  этого нужен либо Python venv (**~4.7 ГБ**, при запуске из исходников),
  либо папка собранного `.exe` (**~4.9 ГБ**) — не оба сразу, выбери один
  способ запуска.
- **GPU опционален, только NVIDIA**: видеокарты AMD/Intel просто
  автоматически откатятся на CPU — полностью рабочий режим, просто
  медленнее на страницу. Смотри раздел [Ускорение на GPU](#ускорение-на-gpu)
  ниже — что нужно самым новым картам (Blackwell / RTX 50-серии) сверх
  обычной установки драйвера.
- **SSD рекомендуется**: модели загружаются в память при каждом запуске, и
  на обычном HDD это заметно медленнее.

## Быстрый старт

Нужен именно **Python 3.11** (ML-зависимости ниже пока не поддерживают
более новые версии).

```
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m app.main
```

Всё, отдельный шаг настройки не нужен. Всё, что нужно приложению, скачается
само при первом переводе страницы, в
`%LOCALAPPDATA%\MRTranslateCache\models\`:

- Детектор текста и модель инпейнтинга (небольшие, ~30–200МБ каждая).
- Собственные веса manga-ocr и RapidOCR, по языку, при первом
  использовании.
- Модель перевода (~2.4 ГБ) — скачивается с докачкой при обрыве соединения
  (не начинает заново), затем конвертируется в более быстрый локальный
  формат (~1 минута, один раз). Если хочешь сделать это заранее, а не во
  время первого перевода, запусти
  `python scripts/convert_nllb_to_ct2.py`.

## Как пользоваться приложением

1. **Открой что-нибудь**: `File → Open Image / Open Folder / Open CBZ...`,
   или просто перетащи файл на окно. Загруженные страницы появятся как
   миниатюры в панели **Pages** слева.
2. **Выбери языки**: выпадающие списки `From`/`To` сверху. У `From` есть
   вариант `Auto-detect`, если не знаешь (или не хочешь указывать) исходный
   язык.
3. **Переведи**: `Translate All` переводит весь набор в фоне с
   прогресс-баром; `Translate This Page` — только ту страницу, которую
   сейчас смотришь.
4. **Проверь результат**: три кнопки над страницей — `Original`,
   `Translated`, `Compare` — переключают между нетронутым сканом,
   переведённой версией и шторкой сравнения обоих вариантов.
5. **Исправь, что выглядит не так**: кликни прямо по облаку на странице
   (или выбери его из списка `Detected Text Boxes` справа), чтобы выделить.
   Ниже появится его исходный текст (OCR) и текущий перевод — отредактируй
   перевод, и картинка перерисуется автоматически. Боксы с пустым или
   низкоуверенным результатом OCR помечены значком ⚠ в списке — понятно,
   что стоит перепроверить.
6. **Добавляй страницы в любой момент**: `File → Add Image(s) / Add Folder /
   Add CBZ...` добавляет к уже загруженному, а не заменяет его. Перетаскивай
   страницы в списке, чтобы поменять порядок; правый клик или кнопки
   `Remove Selected`/`Remove All`, чтобы убрать ненужные страницы.
7. **Экспортируй**: `File → Export as Images...` или `Export as CBZ...`,
   когда результат устраивает.

### Горячие клавиши

| Клавиша | Действие |
|---|---|
| `Ctrl+O` | Открыть картинку |
| `Ctrl+E` | Экспортировать как картинки |
| `Ctrl+0` | Сбросить зум по размеру страницы |
| `←` / `→` | Предыдущая / следующая страница (из списка страниц или просмотрщика) |
| `↑` / `↓` | Предыдущая / следующая страница (пока фокус на просмотрщике) |
| `Space` | Переключить Original ↔ Translated |
| Колесо мыши (на странице) | Зум +/-, по центру курсора |
| Перетаскивание (на странице) | Панорама при увеличении |

### Настройки

`Settings → Preferences...`:

- **OCR Backend**: заставить использовать RapidOCR для японского вместо
  специализированного движка `manga-ocr`, если так удобнее.
- **Device**: Auto (пробовать GPU, откат на CPU) / CPU only / GPU only.
- **Reading Mode**: Horizontal или Vertical — меняет только то, с какого
  края появляются кнопки листания при наведении (слева/справа или
  сверху/снизу); стрелки на клавиатуре работают одинаково в обоих режимах.
- **Custom Font**: шрифт на всё приложение, используется там, где у
  страницы или бокса нет своего собственного шрифта.

## Сборка в отдельный `.exe`

```
pip install pyinstaller
pyinstaller pyinstaller.spec
```

Собирает `dist/MangaReaderTranslate/MangaReaderTranslate.exe` —
самодостаточную папку (несколько ГБ, в основном ML-библиотеки), которая
запускается на целевой машине без отдельной установки Python. Модели
по-прежнему скачиваются в `%LOCALAPPDATA%` при первом использовании, как и
при запуске из исходников.

## Ускорение на GPU

Работает из коробки на большинстве видеокарт NVIDIA, но на самых новых
(Blackwell / RTX 50-серии) нужно больше, чем просто
`pip install onnxruntime-gpu`:

1. `requirements.txt` уже фиксирует `onnxruntime-gpu==1.27.0` и пакеты
   `nvidia-*-cu12` (cublas/cudnn/cufft/curand/nvrtc) — этого достаточно для
   более старых видеокарт (даёт cuDNN 9).
2. **Именно на Blackwell** `onnxruntime-gpu` нужны рантайм-библиотеки CUDA
   **13.x** (`cublas64_13.dll` и т.д.), которых пока нет в виде чистых
   pip-пакетов. Поставь официальный
   [CUDA 13.x Toolkit](https://developer.nvidia.com/cuda-downloads) для
   Windows (большой, ~2.5 ГБ, нужны права администратора) — это заодно
   чинит поддержку GPU в `ctranslate2`, у него та же зависимость от cuDNN.
3. **Перезагрузись после установки Toolkit.** Его компоненты уровня ядра не
   всегда подхватываются без перезагрузки.
4. `app/core/gpu_bootstrap.py` делает всё остальное автоматически при
   запуске — регистрирует и папки пакетов `nvidia-*-cu12`, и системную
   папку `bin\x64` CUDA Toolkit как пути поиска DLL.
5. Проверь `Settings → Preferences → Device` — если инициализация GPU не
   удалась по любой причине, приложение молча откатывается на CPU вместо
   падения.

## Как это устроено

| Этап | Что используется | Почему |
|---|---|---|
| Детекция | [comic-text-detector](https://github.com/dmMaze/comic-text-detector) (ONNX) | Обучен именно на страницах манги/комиксов — находит облака и звуковые эффекты, оценивает поворот/стиль каждого региона по форме текстовой маски. |
| OCR | [manga-ocr](https://github.com/kha-white/manga-ocr) (японский), [RapidOCR](https://github.com/RapidAI/RapidOCR) (всё остальное) | manga-ocr дообучен именно на диалогах манги; RapidOCR (PP-OCRv5, чистый ONNX, без PaddlePaddle) покрывает остальные ~39 исходных языков по типу письменности. |
| Перевод | [NLLB-200-distilled-600M](https://huggingface.co/facebook/nllb-200-distilled-600M) через [CTranslate2](https://github.com/OpenNMT/CTranslate2) | Одна модель покрывает все ~40 языков в обе стороны, квантована в int8 для быстрого локального инференса. |
| Инпейнтинг | [ogkalu/lama-manga-onnx-dynamic](https://huggingface.co/ogkalu/lama-manga-onnx-dynamic) (LaMa) | Справляется с инпейнтингом больших масок с сохранением текстуры намного лучше простого `cv2.inpaint`, который размазывает растровый тон манги. |
| Рендер | Pillow + свободные (OFL) шрифты | Перенос строк + автоподбор размера, поворот под оригинал, стиль по роли (диалог/акцент/SFX/мысли/механический голос). |
| UI | PySide6 (Qt, LGPL) | Десктопное приложение: просмотрщик страниц, панель пакетной обработки, панель ручной правки, настройки. |

  только некоммерческое использование**. Для этого бесплатного личного
  инструмента это нормально, но его нельзя продавать или встраивать в
  коммерческий продукт без замены движка перевода.
- `comic-text-detector` — **GPL-3.0**. Проект только обращается к его
  опубликованным ONNX-весам через `onnxruntime` (исходный код GPL никуда

- LaMa/`lama-manga-onnx-dynamic`, RapidOCR и Noto Sans — **Apache-2.0** /
  **SIL OFL** — никаких ограничений, кроме указания авторства.
- Встроенные шрифты (Comic Neue, Bangers, Caveat, Share Tech Mono, все
  **SIL OFL**) намеренно *не* являются платными шрифтами, стандартными для

  используют без лицензии. Проект сопоставляет те же роли леттеринга
  (диалог/акцент/SFX/мысли/механический голос) со свободными аналогами.

## Известные ограничения

- Автоопределение исходного языка — эвристика по диапазонам Unicode поверх
  первого прохода RapidOCR, а не настоящая модель определения языка:
  надёжно работает для ja/ko/zh/ru/el/ar/hi/th, для остальных по умолчанию
  считает английский. Всегда можно переопределить вручную в интерфейсе.
- Для иврита, бенгальского и кхмерского пока не подключена модель
  распознавания RapidOCR.
- Определение стиля/поворота региона — эвристика по форме текстовой маски
  (компактность + угол), а не обученный классификатор: не может надёжно
  отличить мысль от выкрика, только "наклонено → вероятно SFX" и
  "неровные штрихи → вероятно акцент".
- Авто-разбиение длинных лент ищет однородные/пустые строки-разделители
  между склеенными страницами; полностью бесшовная лента без единого
  зазора останется одной страницей, а не будет разрезана наугад.
